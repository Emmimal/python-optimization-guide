## Why This Repository Exists

Python optimization is not about clever syntax.  
It is about:

- Reducing interpreter overhead
- Choosing the right data structures
- Moving work into compiled code
- Measuring performance correctly

This repository provides **clean, minimal examples** to demonstrate those principles.

- Profiling Python code using `cProfile`
- Algorithmic vs micro-optimizations
- List vs set vs dictionary performance
- Loop optimization and elimination
- NumPy vectorization vs Python loops
